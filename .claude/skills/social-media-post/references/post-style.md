# Post Style Guide

## LinkedIn (long-form)

**Length**: 180–280 words. Long enough to teach, short enough to read in 30 seconds.

**Structure**:
1. **Hook line** — bold question or surprising claim. Pulls people out of the feed.
2. **Setup paragraph** — 2–3 sentences framing the problem.
3. **The pattern** — present the central formula or anatomy.
4. **Color-coded breakdown** — 3–6 bullet points, each starting with a colored emoji (🔵 🟡 🔴 🟢 🟣) matching the infographic.
5. **2 concrete examples** — show the pattern applied to real-world cases.
6. **The "aha"** — the non-obvious takeaway. This is what people will quote.
7. **Practical math/rule** — optional but valuable. Memory math, conversion formula, etc.
8. **Engagement question** — invites comments. Specific beats generic.
9. **Hashtags** — 3–5 max, at the very bottom.

**Voice**: professional but warm. First person. Conversational, not corporate.

**Avoid**:
- "Excited to share..." (cringe)
- Walls of text without bullets
- More than 5 hashtags
- Self-promotion in the first 3 lines

**Example opening lines that work**:
- "Ever wondered what the second number in `Qwen3-30B-A3B` actually means?"
- "Decoding LLM model names: a 60-second guide"
- "The thing nobody explained to me about Mixture of Experts:"

## X / Twitter Thread

**Length**: 5–8 tweets. Each tweet ≤ 280 chars.

**Structure**:
1. **Tweet 0 (hook)** — the thread starter. Must promise value clearly. End with `↓` or `🧵`.
2. **Tweet 1/** — define the core term/pattern. Concrete.
3. **Tweets 2-4** — one concept per tweet. Use examples, not abstractions.
4. **Tweet 5** — the "aha" — the punchline that justifies the whole thread.
5. **Tweet 6 (optional)** — practical math/rule.
6. **Tweet 7 (final)** — call to action. Either "now go read X" challenge or "follow for more" type close.

**Voice**: punchy, terse, almost Twitter-poetry. Use line breaks generously. No fluff.

**Formatting tricks**:
- Number tweets with `1/`, `2/`, etc.
- Use `→` for derivations
- Use `·` for list items (cleaner than bullets on X)
- Use code formatting (`backticks`) for tokens
- One idea per tweet — never cram two

**Avoid**:
- Long paragraphs (X is vertical-scroll, not horizontal-read)
- More than one emoji per tweet
- Generic "Follow me for more!" without specifics
- Hashtags inside the thread (they look spammy on X)

**Example hooks that work**:
- "🧵 How to read any LLM model name in 60 seconds"
- "The thing about MoE models that nobody tells you:"
- "I spent 2 hours decoding LLM names so you don't have to. Here's the cheatsheet:"

## Shared Rules (both platforms)

- **No AI-disclosure preamble** — just deliver the content
- **No "as we discussed"** or other context that doesn't make sense to a stranger
- **Real numbers** — if a model uses 140GB, say 140GB, not "a lot"
- **Concrete > abstract** — name actual models, libraries, versions
- **Don't repeat the same example in both** — vary examples across platforms so following both rewards the reader
